# SaaS Example

## Raw Fact Pattern

- Web app with auth, billing hooks, and team settings
- Dashboard plus recurring workflow automation
- Docs mention admins and reporting

## Good Framing

- User value: replaces spreadsheet-heavy recurring ops work with a repeatable workflow
- Commercial framing: there is a visible path to paid convenience and team usage
- Risk framing: PMF and retention are not proven just because billing code exists

## Avoid

- Treating incomplete billing integration as a monetized business
- Treating every dashboard widget as a meaningful differentiator
