# Product Value Rubric

Use this rubric to convert repo facts into user value.

## Extract

- Target user
- Core scenario
- Current workaround or legacy workflow
- Main friction in that workflow
- What the repo removes, shortens, automates, or improves
- Direct gains: time saved, complexity reduced, confidence improved, visibility improved, quality improved

## Questions

1. Who feels the pain often enough to care?
2. What are they doing today instead?
3. Where is the friction: setup, switching cost, waiting, coordination, repetition, uncertainty?
4. What becomes easier after adopting this project?
5. Why would someone keep it in their workflow instead of trying it once?

## Output Pattern

- `Target user`: be specific enough to be commercially useful
- `Pain`: describe the costly or annoying part of the old workflow
- `Replacement`: explain what the repo replaces or simplifies
- `Benefit`: state the direct user-facing gain

## Guardrails

- Do not claim quantified ROI unless the repo or user provides evidence
- Do not confuse "supports many features" with "solves an urgent pain"
- Prefer one concrete use case over a vague multi-persona summary
