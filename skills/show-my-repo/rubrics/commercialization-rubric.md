# Commercialization Rubric

Use this rubric to turn repo evidence into commercially legible positioning.

## Narrative Frames

### Problem -> solution -> value

- What is the old way?
- Why is it slow, fragmented, expensive, or unreliable?
- What does this project compress or simplify?
- Why would someone try it now?

### Differentiation

Compare against:

- manual workflow
- generic tools
- adjacent competitors
- technically similar tools with a worse user path

### Monetization logic

Look for signals of:

- paid convenience
- team features
- hosted service potential
- premium integrations
- workflow depth that could justify subscription

If the repo does not support a monetization claim, say the charging path is unclear.

### Founder or team credibility

Infer only from evidence such as:

- architecture judgment
- clean scope control
- documentation quality
- deployment discipline
- test and release hygiene

## Guardrails

- Do not call a technical choice a moat unless it creates a durable product advantage
- Do not pretend the repo has PMF, traction, or revenue validation
- Distinguish clearly between a credible commercialization path and a proven business
