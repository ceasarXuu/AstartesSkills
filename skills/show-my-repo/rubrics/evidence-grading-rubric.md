# Evidence Grading Rubric

Use this rubric on every major claim.

## Strength Labels

### Strong

- Docs state it clearly
- Code or config clearly implements it
- Supporting scripts, tests, or demos align with the claim

Safe language:
- implemented
- clearly supported
- explicit in docs and code

### Medium

- Code strongly supports the conclusion
- Docs are partial or indirect
- The claim is reasonable but not directly stated

Safe language:
- inferred from implementation
- likely designed for
- code strongly suggests

### Weak

- Only partial evidence exists
- The conclusion depends on interpretation
- Important support is missing

Safe language:
- possible
- early indication
- should not overclaim

### Planned

- Roadmap, TODOs, or docs indicate future work
- Current implementation is absent or partial

Safe language:
- planned
- roadmap item
- not yet implemented

## Acceptable Evidence Sources

- README and docs
- directory structure and module boundaries
- core implementation files
- configs and dependency declarations
- tests and CI
- demo assets and example flows
- issues, changelogs, and roadmap notes

## Prohibited Behavior

- Presenting a weak inference as a shipped feature
- Presenting a roadmap item as current product capability
- Presenting third-party capability as proprietary differentiation
