# Highlights Rubric

Use this rubric to decide what is worth emphasizing.

## Highlight Categories

### Product highlights

- clear use case
- low-friction workflow
- fast time to value
- strong demo path

### Technical highlights

- clean modularity
- real local-first, privacy, realtime, or orchestration design
- reliability or observability choices
- extensibility or maintainability that matters to product delivery

### Commercial highlights

- clear target user
- natural paid upgrade path
- differentiated entry point
- sharing or community spread potential

### Execution highlights

- coherent docs
- scripts and tooling that reduce setup friction
- tests or CI that increase credibility
- evidence of iteration discipline

## Difficulty Categories

- product: user education, fuzzy scope, workflow adoption
- engineering: realtime, sync, auth, long-running tasks, cost control, privacy boundaries
- commercialization: distribution, monetization, crowded market, unclear wedge

## Priority Labels

- `P1`: must say; central to audience memory
- `P2`: useful, but supporting
- `P3`: technical backing only; do not lead with it

## Selection Rules

- Prefer memorable user value over impressive-sounding internals
- Prefer evidence-backed claims over speculative differentiation
- A point can be technically interesting and still be a poor external highlight
