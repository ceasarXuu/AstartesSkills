# Investor Version Template

## Structure

1. What direction this project sits in
2. Which painful workflow it improves
3. Why existing approaches are weak or fragmented
4. What the repo has already implemented
5. Why the implementation suggests execution strength
6. What the likely commercialization path is
7. What stage it is actually in now

## Length

150 to 300 words.

## Tone

- commercial but restrained
- explicit about stage
- uses technical details as credibility, not as a dump
