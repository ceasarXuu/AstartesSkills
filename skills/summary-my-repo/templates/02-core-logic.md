# 02 Core Logic Template

Write to:

- `summary-my-repo/YYYY-MM-DD-vN/02-core-logic.md`

## Suggested Sections

1. Core files table
2. End-to-end workflow walkthroughs
3. Important control flow or data flow
4. Key invariants and assumptions
5. Current sharp edges or limitations

## Rule

For each core file or workflow block:

- explain what it does, what it depends on, and what breaks if it changes incorrectly
- reference one or more snippet ids from `03-code-evidence.md` as proof

- Role:

- Outputs:
- Invariants:
- Evidence snippets: `S01`, `S03`
- Why this block matters:
